# 운동 기록 앱

개인용 4일 스플릿(월·화·목·토) 운동 기록 웹앱입니다. 데이터는 폰 브라우저의 `localStorage`에 저장되며, 서버나 로그인이 필요 없습니다.

## 처음 배포하기 (5분)

1. **GitHub에서 새 저장소 만들기**
   - github.com 로그인 → 우측 상단 `+` → `New repository`
   - Repository name: `workout-log` (원하는 이름으로 변경 가능)
   - Public으로 설정 (Private이면 Pages 사용 시 유료 플랜 필요할 수 있음)
   - `Create repository` 클릭

2. **파일 업로드**
   - 저장소 페이지에서 `Add file` → `Upload files`
   - 이 폴더의 `index.html` 파일을 드래그해서 업로드
   - 하단 `Commit changes` 클릭

3. **GitHub Pages 켜기**
   - 저장소의 `Settings` 탭 → 왼쪽 메뉴 `Pages`
   - `Branch`를 `main` (또는 `master`), 폴더는 `/ (root)`로 선택 → `Save`
   - 1~2분 기다리면 상단에 `https://<내아이디>.github.io/workout-log/` 형태의 주소가 생성됨

4. **아이폰에 설치**
   - 위 주소를 아이폰 사파리로 접속
   - 공유 버튼(⬆️) → `홈 화면에 추가`
   - 홈 화면 아이콘을 눌러 접속하면 앱처럼 열림

## 나중에 업데이트하기

내용(운동 종목, 세트 수, 무게 목표 등)을 바꾸고 싶을 때:

1. 저장소에서 `index.html` 파일 클릭 → 연필 아이콘(edit) 클릭
2. `ROUTINES` 객체 부분에서 운동 이름·세트·횟수·무게를 원하는 대로 수정
3. 하단 `Commit changes` 클릭
4. 1~2분 후 배포된 사이트에 자동 반영됨 (앱을 새로고침하면 보임)

Claude에게 "index.html 파일에서 화요일 루틴에 OO 운동을 추가해줘" 라고 요청하면, 수정된 파일을 다시 만들어드릴 수 있어요. 그 파일을 다시 업로드(덮어쓰기)하면 됩니다.

## 데이터 백업 (중요)

이 앱은 폰 브라우저에만 데이터를 저장하기 때문에, 브라우저 데이터를 지우거나 기기를 바꾸면 기록이 사라질 수 있습니다.

- 앱 하단의 **내보내기** 버튼을 가끔 눌러서 백업 JSON 파일을 저장해두세요.
- 기기를 바꾸거나 데이터가 사라졌을 때는 **가져오기** 버튼으로 백업 파일을 불러오면 복원됩니다.

## 여러 기기에서 동기화하고 싶다면

지금 방식은 폰 한 대에만 저장됩니다. 아이패드 등 다른 기기와도 기록을 공유하고 싶어지면, Firebase나 Supabase 같은 무료 백엔드를 연결해 실시간 동기화를 추가할 수 있습니다. 필요할 때 말씀해 주세요.
